| **唯一规范名** | 推荐 | 在全文内可检索的标准称呼；同名口语必须区分场景 |
| **类型** | 推荐 | `指标` / `维度` / `规则` 之一 |
| **定义** | 是 | 一句话说明「是什么」 |

### 2. 规则 R02：B 端 / C 端学生（`role` + `attribution`）

- **唯一规范名**：规则 R02：B 端 / C 端学生（role + attribution）
- **类型**：规则
- **定义**：学生用户需区分 B 端 / C 端时，**禁止**仅用 `role='student'`。
- **计算方式**：B 端：`role='student'` 且 `attribution in ('b','B端用户')`；C 端：`role='student'` 且 `attribution not in ('b','B端用户')`；新注册 B 端等变体见「学生指标口径」逐条。
- **表来源**：`dw.dim_user` 及条目所列其它表

### 3. 规则 R03：教师默认全量；标注 B 端 / C 端

- **计算方式**：C 端教师：`role='teacher'` 且 `attribution not in ('b','B端用户')`；三方平台教师等见「教师指标口径」。
- **表来源**：`dw.dim_user`

### 8. 规则 R08：数仓分层、血缘与 DDL 枚举写作约定（与【平台】规则 R34 对齐）

- **唯一规范名**：规则 R08：数仓分层、血缘与 DDL 枚举写作约定（与【平台】规则 R34 对齐）
- **类型**：规则
- **定义**：公司数仓按**四层**组织；数据**自上而下加工**（上游定义语义，下游继承并叠加汇总/筛选）。编写或阅读 `code/sql/表结构`、看板 SQL 时，应先用**表所在层级**判断字段/指标应在哪里写清定义与枚举，**避免**在每一层把同名含义再写一遍。（分层细则与【平台】一致，详见【平台】_shihua.md **规则 R34** 全文。）
- **计算方式**（层级职责与典型 schema，用于推理血缘，而非替代具体 DDL）：
  1. **ODS（落地层）**  
     - 承接业务库**原始镜像**，与业务侧实体**同构**，原则上不做业务口径转换。  
     - 典型：`ods*` / `ods_rt.*` 等；仅作明细来源，**不作为**报表与指标定义的权威落点。
  2. **DW D / DW 明细层（`dw`）**  
     - 对 ODS 做清洗、格式统一、脱敏等；保留**业务最细粒度**一行一事实（粒度以具体表为准）。  
     - **命名习惯**：维表 `dim_*`，事实表 `fact_*`；周期快照表常用 `_day` / `_week` / `_month` 后缀。  
     - **语义权威**：码表、枚举、主数据含义优先在本层或贴源维表中**集中说明**；下游同源同名字段默认**继承**，除非 DDL 另有「重定义」说明。
  3. **DWS 公共汇总层（`dws`）**  
     - 主题宽表与多实体关联，沉淀可复用汇总指标；**命名**常见 `topic_*`。  
     - **语义权威**：主题级派生字段在本层 DDL/字典写清；底层码值若仅透传，可与 `dw` 共用解释，**不重复**穷举全表枚举。
  4. **AWS 数据集市 / 应用层（`aws`）**  
     - 面向场景与看板的应用表；**本层新增**的聚合与归因口径在此说明；字段明显来自 `dw`/`dws` 且含义未变时**默认继承**上游，DDL 写清关联键与粒度即可。
- **依赖与使用约束**（推理血缘与排错）：`dws`/`aws` **不**为常规报表直连 ODS；ODS **不得**反向依赖上层；同层 `dws` 长链需谨慎。
- **两条标准数据路径**：稳定业务 **ODS → dw → dws → aws**；探索需求可走 **ODS → dw → aws**，仍遵守跨层边界。
- **写作约定（本仓库，与【平台】_shihua.md 规则 R34 一致）**：  
  - 某字段在**最上游首次出现**且带业务枚举/口径时，在**该层对应表**的 DDL 或本 glossary **写全**；下游同名、同含义字段用「见 `schema.table` 字段 xxx」或「继承 `dw`/`dws` 某表定义」一笔带过。  
  - 若下游对同名字段**收窄、重算或改义**（例如过滤条件变化、聚合后不再可比），必须在**当前表层**显式写出差异，避免读者误用上游枚举。
- **表来源**：`ods*` / `ods_rt.*`、`dw.*`、`dws.*`、`aws.*`（以 `code/sql/表结构/【智课】_shihua` 与 `code/sql/表结构/【平台】_shihua` 实际归档为准；**跨域订单/用户类枚举**常以平台主题宽表为权威，智课 DDL 第三段用「见某表」承接）。
