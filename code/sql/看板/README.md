# 看板 SQL 收集

存放 **BI / 数据看板 / 报表** 在配置层使用的**固化 SQL**（或从工具导出的等价脚本），与「临时探查」区分，便于口径追溯与复用。

## 与 `sql-patterns.md` 的关系

| 类型 | 位置 | 说明 |
|------|------|------|
| 短模板、可参数化片段 | `.cursor/knowledge/sql-patterns.md` | `${param}` 占位，强调可复用 |
| 完整看板查询（常含固定日期、多段 CTE） | `code/sql/看板/【域】_维护人/*.sql` | 与真实看板一一对应或按图表拆文件 |

新增看板后：**在此目录落盘 SQL**，并在 `sql-patterns.md` 的索引表中增加一行链接（可选：只链文件路径）。

## 目录结构（与表结构域一致）

```
code/sql/看板/
├── 【电销】_tanchen/
├── 【投放】_xinyu/
├── 【新媒体】_huihui/
├── 【平台】_shihua/
├── 【智课】_shihua/
└── 【宽表】_xingling/
```

## 文件命名建议

`{看板简称或ID}_{图表或区块简述}.sql`  
示例：`经营日报_核心指标.sql`、`电销产能_日汇总.sql`

## 每个 SQL 文件建议表头（复制到文件顶部）

```sql
-- =====================================================
-- 看板名称：{在 QuickBI / 网易有数 / Metabase 等中的名称}
-- 业务域：【宽表】_xingling（与目录后缀一致）
-- 图表/组件：{若是多 SQL 看板，写明本文件对应哪一块}
-- 工具与位置：{产品名 + 工作空间/报表路径，可写「内部文档链接」}
-- 刷新周期：{T+1 / 小时级 / 实时 等}
-- 维护人：{姓名}
-- 关联指标：→ glossary/【域】_维护人.md#{术语锚点}
-- 主用表：→ code/sql/表结构/【域】_维护人/{表}.sql
-- 最后同步自看板日期：YYYY-MM-DD
-- ⚠️ 注意：{口径特殊点、与线上一致性检查方式}
-- =====================================================
```

正文为**可从工具中直接复制**的 SQL；若工具使用图形建模无 SQL，可改为「逻辑说明 + 导出 SQL / 伪代码」同文件用注释分隔。
