# 归因分析框架

> 适用场景：C 端业务指标异动（GMV / 付费用户 / 转化率 / ARPU 等）  
> 完整 SOP 见：[attribution-analysis-guide.md](attribution-analysis-guide.md)  
> 下钻逻辑详解见：[q3-drilldown-logic.md](q3-drilldown-logic.md)

---

## 一、六问框架总览

```
Q1 异动精确化   明确指标 + 选对比基准 + 与业务方校准异动标准
      ↓            不满足条件 → 记录后关闭，不进归因
Q2 真伪判定     数据质检，排除采集/口径问题
      ↓            不通过 → 终止，修复数据后重跑
Q3 定位下钻     全量/局部判断 → 多维对比 → 专项深挖
      ↓            定位：哪里跌
Q4 假设验证     Q3 对比结论 → 验证显著维度
      ↓            验因：为什么跌（Q3 结论即 Q4 假设来源）
Q5 主因判定     一级拆解 + 贡献度量化
      ↓
Q6 建议输出     [数据事实 + 全量/局部定性] + [发现问题] + [具体行动]
```

## 二、Q1 对比基准选择

| 场景 | 基准 |
|---|---|
| 日常监控 | 近四周同星期均值 |
| 强季节性指标 | 同比去年同周 |
| 紧急排查 | 环比上周 |
| 大促后复盘 | 去促基线 |

**异动标准**：动态阈值 μ ± σ，取近 28 天日粒度数据自动计算，大促日先剔除再建基线。当前周均值落在区间外才触发归因，落在区间内记录关闭。有已知业务原因先排除，无已知原因再进归因。

## 三、Q3 下钻逻辑

### 第一步：全量 vs 局部

扫描四个维度的变化方向：**学段 / 渠道 / 用户类型 / 时段**

- **有逆势维度**（某维度涨而其他跌）→ 局部问题，进第二步
- **所有维度同向跌** → 全量问题，看时段分布和同比，找退潮节点，不急于下钻

### 第二步：多维对比（局部问题专用）

用"对比定性"，找哪里和其他维度明显不一样：

**A. 年级 × 渠道**（最优先，一张交叉表同时读两个维度）

| 发现 | 定性 |
|---|---|
| 某学段所有渠道都跌，其他学段正常 | 学段个性问题 |
| 所有学段某渠道都跌，其他渠道正常 | 渠道个性问题 |
| 只有某学段 × 某渠道跌 | 交叉特有问题 |
| 所有学段所有渠道都跌 | 回第一步，重新判断是否全量 |

**B. 用户分层**

| 用户层 | 跌了说明 |
|---|---|
| 新用户首购 | 获客/转化链路问题 |
| 老用户续费 | 产品/服务质量问题 |
| 高净值用户 | 高价课/套餐策略问题 |
| 流失召回 | 私域运营/促销策略问题 |

**C. 时间分布**（辅助，跑日粒度趋势图）

- 某几天骤跌，其他天正常 → 系统故障 / 推送中断 / 投放暂停
- 每天持续跌，幅度接近 → 结构性问题，非运营动作导致

**可选：新老用户 × 学段交叉**

A + B 看完仍然模糊时补跑。初中新用户跌 + 老用户正常 → 初中获客链路问题，非产品问题。

### 第三步：专项深挖（锁定跌点后才进）

| 跌点 | 专项 |
|---|---|
| 学段有问题 | SKU 下钻：`good_name × mid_grade`，找 TOP 下跌 SKU，判断季节性/曝光/价格/内容 |
| APP/商业化有问题 | 四层漏斗：曝光 → 点击 → 试听 → 支付，找断点层；拆 iOS/Android |
| 电销有问题 | 团队三步：团队×学段拆 GMV → 线索量 vs 转化率 → 人效对比 |

## 四、Q5 一级拆解公式

| 指标 | 拆解 |
|---|---|
| GMV | 付费用户数 × AOV |
| 转化率 | 多层漏斗逐层拆，找断点 |
| 线索量 | 触达人数 × 留资率 |
| ARPU | GMV / 活跃用户数 |
| 电销 GMV | 线索量 × 转化率 × 客单价 |

先确认哪个因子变了，只追变化最大的那个，不同时往下追所有因子。

---

## 五、Q6 建议输出格式

```
[数据事实 + 全量/局部定性] + [发现问题] + [具体行动]
```

示例：
> [小升初] GMV ↓24.5%（-3.1万，局部问题），4 月初签约高峰结束后自然退潮，建议 4 月底启动专题活动提前拉需求。

> 商业化电商 ↓33.4%（局部问题，其余渠道正常），建议本周内核查商品链接状态与投放计划。

## 六、常见错误

| 错误 | 后果 |
|---|---|
| Q1 不校准就进归因 | 分析波动在正常范围内的指标，浪费资源 |
| 跳过全量/局部判断直接下钻 | 把季节性退潮归因为渠道问题 |
| 多维对比没做完就进专项 | 遗漏逆势维度，结论方向错误 |
| 跳过一级拆解直接下钻维度 | 不知道是用户问题还是 AOV 问题，下钻方向错误 |
| Q3 结论不传递给 Q4 | 重复分析，浪费资源 |
| 建议只写现象不写动作 | 业务不知道怎么办 |
