# Q3 下钻逻辑梳理

> 归属文档：洋葱学园 · 归因分析 SOP v2.1  
> 适用步骤：Q3 定位下钻

---

## 核心思路

**先收窄问题范围，再决定往哪钻**，不是上来就无脑拆维度。

---

## 前置步骤：全站大盘对标

**进内部维度分析之前，先确认异动是行业性的还是本品特有的。**

| 大盘表现 | 判断 | 策略 |
|---|---|---|
| 全站大盘同向下跌 | 环境因素主导 | 优先标记外部原因，内部局部归因降权；重点确认跌幅是否与大盘同步，而非追究内部原因 |
| 全站大盘正常 | 内部问题 | 进入正式下钻流程 |
| 全站大盘上涨、本品下跌 | 内部问题且有竞争风险 | 加急归因，重点查渠道和商品维度 |

> 全站大盘数据来源：行业 GMV 指数 / 平台 DAU 趋势 / 竞品监控报告。无外部数据时，可用平台整体订单量作为替代参照。

**四维并行跑，输出贡献度排序，再决定往哪下钻：**

同时计算年级 / 渠道 / 商品 / 销售团队四个维度的 `delta_gmv`，按绝对值降序排列，贡献度最大的维度优先进入第一步判断。不串行逐维度看，避免遗漏逆势维度。

---

## 第一步：判断问题是全量还是局部

### 1.1 用哪些维度来判断

同时跑以下四个维度的变化率，看有没有**逆势维度**（某维度涨而其他维度跌）：

| 维度 | 具体拆法 | 为什么看这个 |
|---|---|---|
| **学段** | 小学 / 初中 / 高中 | 各学段受季节性影响不同，升学节点也不同；全跌说明不是学段个性问题 |
| **渠道** | 商业化 / 电销 / 入校 / 自然流量 | 渠道之间运营逻辑完全独立；某渠道逆势说明是渠道特有问题 |
| **用户类型** | 新用户 / 老用户续费 / 流失召回 | 新老用户受不同因素驱动；新用户跌不影响老用户说明是获客问题 |
| **时段** | 工作日 vs 周末 / 分时段 | 教育电商周末天然高峰；若只有工作日跌可能是 B 端/电销的问题 |

> 这四个维度是"判断全量/局部"的最小必要集，不需要交叉，只需要看各自的变化方向是否一致。

### 1.2 判断标准

不用固定阈值，看两个信号：

**信号一：有没有逆势维度**

某维度涨而其他维度跌，是最强的"局部问题"信号。比如电销 +3.7% 而商业化 -14.3%，不需要看差距多大，方向相反本身就说明问题是局部的。

**信号二：各维度跌幅是否分化**

所有维度跌幅接近、方向一致，倾向全量问题。但"接近"没有固定数字——和这个指标的历史波动范围比，波动本来就大的指标，各维度差 8% 也可能是正常噪音；波动很小的指标，差 3% 就值得关注。

> 所以判断全量/局部，首先找逆势维度，其次结合历史波动范围感知分化程度，不套固定百分比。

有模糊地带时（各维度方向一致但跌幅差异明显），优先看有没有已知业务原因（如某渠道在做活动），先排除已知原因再重新判断。

**全量问题**：不要急着下钻。大概率是季节性退潮、大促后回落、外部环境，重点看时段分布和同比，找到"退潮节点"就够了。

**局部问题**：进入第二步，用对比定位哪里有个性问题。

---

## 第二步：多维对比锁定跌点（局部问题专用）

**核心思路是"对比定性"，不是"贡献度排名"。**

找的不是"谁贡献最大"，而是"哪里和其他维度明显不一样"——某维度跌但其他正常，就是那里有个性问题。

四个对比维度，按顺序优先级依次看：

### A. 年级 × 渠道（最优先）

跑一张交叉表，同时看学段和渠道两个维度的分化。从这张表可以直接读出两类信息：

**按学段看行**：某学段所有渠道都跌 → 学段个性问题；所有学段某渠道都跌 → 渠道个性问题

**按渠道看列**：某渠道跨所有学段都跌 → 渠道共性问题；某渠道只在个别学段跌 → 交叉特有问题

| 发现 | 定性 |
|---|---|
| 初中所有渠道都跌，小学/高中正常 | 初中学段有个性问题，与渠道无关 |
| 所有学段商业化都跌，电销正常 | 商业化投放问题，与学段无关 |
| 只有初中 × 商业化跌 | 两个维度交叉的特有问题 |
| 所有学段所有渠道都跌 | 重新考虑是否全量问题，回第一步 |

### B. 用户分层（定位流失是哪类用户）

| 用户层 | 跌了说明 |
|---|---|
| 新用户首购跌 | 获客/落地页/转化链路问题 |
| 老用户续费跌 | 产品/服务质量问题 |
| 高净值用户跌 | 高价课或套餐策略问题 |
| 流失召回跌 | 私域运营/促销策略问题 |

## 第三步：专项深挖（锁定跌点后才进）

多维对比做完，跌点已经明确，这时候才进专项。**没有锁定跌点就进专项，是最容易浪费时间的错误。**

### 专项 A：学段有问题 → 下钻到 SKU

按 `good_name × mid_grade` 拉明细，找 TOP 下跌 SKU，然后判断原因类型：

| 原因 | 验证方法 |
|---|---|
| 季节性退潮 | 对比去年同期同 SKU，若同样跌则是自然周期 |
| 曝光骤降 | 查 show UV 有没有骤降（下架/减曝光） |
| 价格调整 | 对比两周 sub_amount 均值 |
| 内容质量 | 查完课率/差评率 |

### 专项 B-1：APP/商业化有问题 → 四层漏斗

```
曝光 UV → 点击 UV → 试听 UV → 支付 UV
```

逐层算转化率，找断点在哪一层。同时拆 iOS vs Android，判断是否单平台问题。

### 专项 B-2：电销有问题 → 团队三步归因

1. 按团队 × 学段拆 GMV，找跌的团队
2. 拆线索量 vs 转化率：线索少是获客问题，线索多转化低是销售/商品问题
3. 对比团队人效，排查话术/激励/培训

---

## 整体决策树

```
Q3 入口
    ↓
全量 vs 局部判断
    ├── 全量问题 → 看时段分布和同比 → 找退潮节点 → 进 Q4
    │
    └── 局部问题
            ↓
        年级 × 渠道对比
            ↓
        用户分层对比（需要时）
            ↓
        公私域 / 渠道横向对比（需要时）
            ↓
        锁定跌点
            ↓
        ┌── 学段有问题 → 专项A：SKU 下钻
        ├── APP 渠道有问题 → 专项B-1：漏斗
        └── 电销有问题 → 专项B-2：团队归因
            ↓
           进 Q4
```
