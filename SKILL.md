# 归因分析 AI 工作流

## 触发条件
用户描述任意业务指标异动时使用，例如：
- "上周9年级转化率跌了"
- "本周客单价下滑了"
- "GMV为什么降了"

---

## 智能体执行流程

### Step 1：向用户收集信息（一次性问完）

**数据库连接（必填）：**
- SSH 跳板机 IP、用户名、密码
- 数据库内网 IP、端口（默认10010）、用户名、密码

**分析参数（必填）：**
- 异动描述（用户自己说的那句话）
- 分析年级/维度（如：九年级、八年级）
- 近4周的时间窗口（每周的起止日期）

**时间窗口格式说明：**
每周需要两种格式：
- int 格式：用于活跃宽表 `day` 字段，如 `20260323`
- 字符串格式：用于订单表 `paid_time`，如 `2026-03-23`

---

### Step 2：克隆并运行

```bash
git clone https://github.com/xuxingling55-sketch/attribution-analysis.git
cd attribution-analysis
pip install -r requirements.txt

python main.py \
  --ssh-host <SSH_IP> \
  --ssh-user <SSH_USER> \
  --ssh-pass <SSH_PASS> \
  --db-host <DB_HOST> \
  --db-port 10010 \
  --db-user <DB_USER> \
  --db-pass <DB_PASS> \
  --grade "<年级>" \
  --anomaly "<异动描述>" \
  --weeks \
    "上周,20260323,20260329,2026-03-23,2026-03-30" \
    "上上周,20260316,20260322,2026-03-16,2026-03-23" \
    "3周前,20260309,20260315,2026-03-09,2026-03-16" \
    "4周前,20260302,20260308,2026-03-02,2026-03-09"
```

`--weeks` 每条格式：`标签,int开始,int结束,str开始,str结束`

---

### Step 3：分析输出数据，生成报告

程序输出5组原始数据表格，智能体逐组分析：

| 步骤 | 数据内容 | 分析重点 |
|------|---------|---------|
| Step 1 | 活跃人数、付费人数、转化率、GMV、客单价 | 确认哪个指标异动，幅度多大 |
| Step 2 | 线索量、线索来源结构 | 线索是否减少？来源是否偏移？ |
| Step 3 | 用户分层占比变化 | 高净值/新用户/老用户结构是否变化 |
| Step 4 | 外呼量、接通率、有效通话率、人效；职场维度 | 电销执行是否出问题 |
| Step 5 | 商品类型销售结构、各品类客单价 | 是否组合品/高价品销售下滑 |

根据数据变化幅度，输出归因报告，格式：

```
## 归因分析报告：<异动描述>

### 根因定位
🔴 主因：...（贡献度 ~X%）
   证据：...
🟡 次因：...
⚪ 排除：...

### 建议行动
1. ...
2. ...
```

## SQL 口径说明（洋葱学园数仓）

| 字段 | 规则 |
|------|------|
| 活跃宽表时间 | `day` 是 int（如 20260323），用整数区间 |
| 订单表时间 | 用 `paid_time`，不用 `add_time_day` |
| 年级字段 | 用 `mid_grade`，不用 `grade` |
| 正价订单 | `sub_amount >= 39` |
| C端活跃必筛 | `product_id='01'`, `client_os IN ('android','ios','harmony')`, `active_user_attribution IN ('中学用户','小学用户','c')`, `is_test_user=0` |

换用其他数仓时，修改 `src/queries.py` 中的表名和字段名。
